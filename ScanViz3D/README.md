# ScanViz3D: 扇扫数据可视化工具 (Fan Scan Inspector)

## 📖 项目简介
本项目是一个轻量级的 3D 数据可视化工具，旨在读取存储扇扫数据的 `.db` (SQLite) 数据库文件，并通过 Open3D 进行三维点云渲染。
项目的最终目标是打包为独立的 `.exe` 可执行文件，方便在进行复杂的路径规划任务前，快速检查和预览现场的扫描数据质量。

---

## 🧪 开发目标 (实验性)
**本项目是一个 "AI 驱动的 TDD 开发" 实验场。**
核心目的是练习在全 AI 辅助代码生成（使用 Trae）的过程中，通过严格的 Prompt 工程控制代码质量和逻辑稳定性。

### 🤖 AI Agent 协作协议 (The Protocol)
**所有参与本项目开发的 AI 必须严格遵守以下原则：**

1.  **测试驱动 (Test-First / Strict TDD)**
    * **严禁裸奔**：任何功能函数 (`method`) 的编写或修改，必须**同步**或**提前**编写对应的测试用例。
    * **覆盖率要求**：测试必须严格，覆盖正常输入、边界条件（如空数据、异常格式）及错误处理。

2.  **回归安全 (Regression Safety)**
    * **不破坏历史**：新的代码变更严禁导致现有的测试用例失败。
    * **修改原则**：如果必须修改旧逻辑，必须先更新对应的测试代码并解释原因。

3.  **文档即记忆 (Context Maintenance)**
    * 项目根目录下维护 `docs/ai_context.md`。
    * 每增加一个核心功能或类，必须更新该文档中的“函数签名”和“项目树结构”，以便 AI 下次读取时能快速理解上下文。

4.  **环境与栈 (Tech Stack)**
    * **OS**: Windows (PowerShell)
    * **Language**: Python 3.x
    * **Core Libs**: `Open3D` (可视化), `sqlite3` (数据读取), `pytest` (测试), `PyInstaller` (打包)
    * **Note**: 暂时不涉及 C++ 代码。
